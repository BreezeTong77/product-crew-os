# Product Crew OS v0.1.4

发布日期：2026-07-12

## 这次改了什么

产品的主流程从“由教练在文本里记住顺序”改成了 LangGraph 控制图。图会明确暂停在两件不能跳过的事上：

1. 等待真实评审回调。
2. 等待用户确认是否继续。

图中的节点顺序：

```text
输入范围判断
-> 检索证据检查
-> Stage / SOP / Skill 路由
-> Skill 执行证据检查
-> 写 Artifact
-> 构建完整评审包
-> 外部评审暂停
-> 用户决策暂停
-> 写项目记忆
```

## 关键保护

- 没有真实 Skill 执行回执时，artifact 可以保存为草稿，但 Gate 必须阻塞。
- 缺少完整 persona packet、raw review、签名 delegate proof 或用户确认时，不能通过 Gate。
- 只伪造 `runtime_agent_id` 不能冒充真实子 Agent 调用。
- 真实 embedding 可以在输入范围门辅助识别产品任务；要求真实 embedding 时，没有 provider、model、来源引用的检索结果会在路由前停止。
- `runtime_nickname` 仍只做审计信息，不能替换配置中的角色身份；raw review 必须可见。

## 还没有宣称完成的事

Ruby 的 Coze Bridge、OCR/RAG 与旧 CLI 还在迁移兼容层，尚未全部改写成 Python/LangGraph adapter。它们不会被删掉或被假称为已迁移；对应功能要在 LangGraph 中补齐同级 Gate 和回归后再替换。
