# Product Crew OS v0.2.2

发布日期：2026-07-12

## 本次修复的三件事

1. **RAG 必须真的建好才能进入标准 SOP。**
   `rag-bootstrap` 会用本地 BGE 建立 44 SOP 索引。只装了 embedding 包、只用了 hash smoke，或没有索引，都只能是 `runtime_degraded`。

2. **主 Skill 不能悄悄换成别的 Skill。**
   某个 Skill 的辅助脚本因输入不足不能运行时，Runtime 会把同一份 `SKILL.md` 交给已部署的本地 Ollama 执行；不会直接用模板，也不会换成无关的 fallback 冒充成功。

3. **子 Agent 身份要和角色绑定。**
   私有 Coze 绑定文件除了 Bot ID，还需要填写 `approved_runtime_agent_ids`。新的 Delegate Signer 会校验真实运行 ID、Coze 调用 ID、完整 Context Packet 状态后签名；不在对应名单内的评审就是 `invalid_for_gate`。

## 已验证

- 本地 BGE 真实建立 44 SOP 索引，图内检索返回路由候选。
- 本地 Ollama 真实执行 `scope-cutting`，并在 `product-discovery` 脚本缺少结构化输入时回退到同一份 Skill 的模型执行。
- 伪造 Skill 成功、伪造 embedding 命中、未签名回调、绑定角色不匹配的回调都会被拦截。
- 包校验、LangGraph E2E、Python adapter E2E、Delegate Signer HTTP E2E、50 条发布门禁全部通过。

## 仍需用户部署

Figma/Pencil MCP 仍需要真实连接和写入授权；Coze 子 Agent 仍需要填写真实 Bot ID、运行 ID 允许名单，并把 Call Bot 返回先经过已部署的 Delegate Signer。未部署时系统必须保持 `runtime_degraded`，不能模拟通过。
