# Product Crew OS v0.2.3

发布日期：2026-07-12

## 这次解决什么

以前即使流程跑完，也很难回答三个很实际的问题：SOP 有没有找对？Skill 有没有真的跑？子 Agent 有没有真的给回意见？

现在每个项目都会生成一页中文运营指标，按 Runtime 的真实记录计算：

- **SOP 确认命中率**：只统计用户明确确认或纠正过的路由；没有反馈的路由不算正确。
- **Skill 真执行率**：只有 LangGraph 实际执行并持有签名回执的 Skill 才算成功。
- **子 Agent 有效回调率**：只有真实调用、角色绑定、原文评审和签名都通过的回调才算完成。

## Bad Case 怎么回流

- 用户纠正 SOP，会自动生成路由 Bad Case。
- Skill 跑不起来、输出为空或回执不合法，会自动生成 Skill Bad Case。
- 子 Agent 回调缺失、身份不符或签名不合法，会自动生成评审 Bad Case。
- 相同 SOP 纠正累计达到阈值，系统会生成一条待人工确认的调参建议。

系统不会自动调高或调低路由阈值，也不会把未确认路由算成命中。先有证据，再由人确认调整。

## 已验证

- 运营指标 E2E：确认的 SOP、图内 Skill 回执、有效评审回调都分别进入正确指标。
- 连续三次相同 SOP 纠正会形成 Bad Case，并生成待人工确认的调参建议。
- 指标同时写入 Runtime SQLite 与 Obsidian 项目包的 `运营指标/` 目录。
