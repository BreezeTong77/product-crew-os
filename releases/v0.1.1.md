# Product Crew OS v0.1.1

发布日期：2026-06-28

## 本次重点

`v0.1.1` 是 Product Crew OS 的 GitHub 发布版本。

这一版把 Product Crew OS 从“可迁移规则包”推进到“用户 clone 后可以完整理解、检查和部署的 AI 产品教练 Skill 系统”：

- 更清楚地表达产品定位：一间给产品经理用的 AI 产品办公室。
- 更明确地表达技术形态：Workflow-first AI Product Harness。
- 补齐 44 个 SOP 卡片和 Stage -> SOP -> Skill -> Stakeholder -> Artifact -> Stage Gate 闭环。
- 内置 49 个第三方 PM skill / 工作包，降低新用户首次部署成本。
- 明确子 Agent 真实调用、模拟视角、Context Packet 和长期记忆反哺边界。
- 增加 Semantic Stage Router 的技术路线，为后续意图识别、轻量检索、RAG 和长期反馈学习做准备。
- 增加本地 SQLite Runtime、`record-turn` adapter 和 Obsidian-compatible 持久项目包导出，让项目记忆、评审项、Artifact 和事件指标能落到真实数据层。

## 适合谁

- 刚入行、需要资深 PM 带着走流程的产品经理。
- 独立 PM、创业者、AI 产品工作流用户。
- 希望拥有一支可配置 AI 产品团队的人。
- 希望把项目推进过程沉淀为可复用 Artifact Workspace 的团队。

## 调整

- README 从规则说明书改为 GitHub 产品首页。
- 首屏突出：痛点、适用用户、Start Here、团队角色、能力地图和本地质检。
- 明确 Product Crew OS 不是多 Agent 群聊，而是主控产品教练驱动的产品工作流系统。
- 44 个 SOP 卡片升级为 8 字段结构：输入、默认 SOP、能力调用、输出 Artifact、子 Agent 调用、Review Loop、Stage Gate、下一阶段。
- Skill Router 调整为“内置能力优先，用户自有 skill 可覆盖，模板兜底不断流”。
- 补齐 `stage-boundary-matrix.md` 与 `stakeholder-boundaries.yaml` 的关键阶段触发边界。
- 原型流程明确为：image 概念图 -> HTML Demo -> 可选 Pencil / Figma 承接。
- `agents/openai.yaml` 从只读策略调整为“项目初始化后自动写 Project Runtime”，同时保留用户偏好、产品规则和真实团队材料的显式授权边界。

```text
ruby product-crew-os-skill/tests/validate-package.rb
ruby product-crew-os-skill/tests/run-regression.rb --mock-delegate --check-only
ruby product-crew-os-skill/tests/run-runtime-smoke.rb
ruby product-crew-os-skill/tests/run-sop-e2e-smoke.rb
```

预期输出：

```text
validate-package: PASS
run-regression: PASS
run-runtime-smoke: PASS
run-sop-e2e-smoke: PASS
```

本版本应同时满足：

- 44 张 SOP 卡片字段完整。
- 44 个 canonical stage 均有 router 覆盖。
- Skill dependency registry 覆盖 stage 0-43。
- Bundled skill index 指向真实存在的内置 skill 目录。
- 子 Agent 真实调用 ledger、模拟视角降级、memory snapshot 和 memory delta 回归场景通过。
- 44 个 SOP prompt case 能通过 `record-turn` 写入 SQLite、artifact、Context Packet、Invocation Ledger、Review Item 和 Obsidian 导出。
- 事件表中能看到 `stage_detected`、`skill_selected`、`memory_snapshot_built`、`agent_summoned` 和 `stage_gate_decision`。

## 不包含

- 真实项目 workspace。
- 用户偏好记忆。
- 具体项目材料。
- 内部测评调用记录。
- 外部 SaaS 产品实现。
- 未授权的 Figma、Pencil、Jira、飞书、Notion、Canva 或其他 MCP 写入能力。

## 升级建议

如果你已经使用 `v0.1.0-alpha`：

1. 用本版本完整替换 `product-crew-os-skill/`。
2. 不要只替换 `SKILL.md`，因为 `config/`、`references/`、`templates/`、`tests/` 和 `third_party/skills/` 都是能力的一部分。
3. 保留自己的 Project Workspace 和 User Preference Memory，不要把它们复制进公共 release 包。
4. 如果你有自己的团队 SOP 或 skill，可以作为 overlay 覆盖默认路由。
