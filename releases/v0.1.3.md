# Product Crew OS v0.1.3

发布日期：2026-07-11

## 这版的重点

这版不是增加一堆新概念，而是把现有流程变得更可信：**不能让“看起来跑了”被当成“真的完成了”。**

## 核心更新

- Stage Gate 需要真实 Skill 执行证据、完整 Required 评审和用户确认。主控不能自行把项目推进到通过。
- 模板输出统一标为 `template_degraded`。它可以给用户当草稿，但不能作为 Skill 成功或 Gate 通过的证据。
- Required / Triggered 角色从已保存的路由决策中派生，调用方不能临时删掉必需角色。
- 子 Agent 缺少完整 Persona Context Packet、运行 ID 或原始评审记录时，只能算建议，不能用于 Gate。
- 本地资料接入新增 RAG 证据绑定。低置信 OCR、未授权或未索引来源会阻塞最终 Gate。
- 图片 OCR 采用本地优先：PaddleOCR 为主路径，Tesseract 为 fallback；缺少引擎时明确返回不可用。

## README 重写

README 保留产品最核心的两张图和规则：

- 主控教练如何把用户输入推进成 SOP、Skill、评审、产物和 Gate。
- 项目资料、用户偏好、角色风格和公共规则如何隔离。
- 真实子 Agent、真实 Skill、RAG 证据和用户确认各自的边界。

其余长篇技术解释、面试表达和未来路线移出 README，改由具体参考文档承接。

## 已知限制

- 真实子 Agent 调用依赖宿主平台的 delegation 能力。
- 外部 Skill、MCP、Embedding 和向量库依赖本地环境；不可用时必须显示受限状态。
- 44 SOP 的覆盖证明的是路由和最小链路，不等于每个 SOP 都已完成深度真实业务验证。
