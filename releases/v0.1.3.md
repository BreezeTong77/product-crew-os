# Product Crew OS v0.1.3

发布日期：2026-07-11

## 这版解决什么

`v0.1.3` 重点解决一件事：不能让“看起来跑了”被当成“真的完成了”。

## 更新内容

- Stage Gate 只接受真实执行证据、完整评审证据和用户确认；主控不能自行把项目推进到通过。
- Skill 需要记录执行状态。模板输出会标为 `template_degraded`，可以当草稿，但不能作为通过 Gate 的证据。
- Required / Triggered 角色从已保存的路由决策中派生，调用方不能临时删掉必需角色。
- 子 Agent 缺少完整 Persona Context Packet、运行 ID 或原始评审记录时，结果只能算建议，不能用于 Gate。
- 新增本地资料接入和 RAG 证据绑定：文本、图片和截图可接入项目库；低置信 OCR、未授权或未索引来源会阻塞最终 Gate。
- 图片 OCR 采用本地优先：PaddleOCR 为主路径，Tesseract 为 fallback；没有 OCR 引擎时明确返回不可用。
- README 收紧为产品定位、能力、边界和验证方式，移除面试表达、设计过程和过多实现细节。

## 已验证

以下本地测试已通过：

```text
validate-package
run-regression
run-runtime-smoke
run-sop-e2e-smoke
run-review-loop-e2e
run-loop-50-cases --release-gate
run-source-ingestion-runtime
run-coze-runtime-bridge-smoke
```

其中 OCR 冒烟为本机本地 PaddleOCR 实测。其他测试验证本地规则、运行时写入和门禁，不代表线上用户效果。

## 已知限制

- 真实子 Agent 调用依赖宿主平台的 delegation 能力。
- 外部 Skill、MCP、Embedding 和向量库依赖本地环境；不可用时必须显示受限状态。
- 44 SOP 的覆盖证明的是路由和最小链路，不等于每一个 SOP 都已完成深度真实业务验证。
