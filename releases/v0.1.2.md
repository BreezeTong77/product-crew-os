# Product Crew OS v0.1.2

发布日期：2026-07-05

## 本次重点

`v0.1.2` 是对 `v0.1.1` 的运行能力收口版，重点解决“流程不是光写在文档里，而是能被验证、被追溯、能被用户看见评审依据”的问题。

- 明确结构化评审闭环：从发起 Review 到角色独立评审、raw review 落盘、冲突矩阵、复评范围收敛、用户确认退出。
- 将子 Agent 的评审结果变成可追溯资产：`review-session.md`、`review-items`、`conflict-matrix`、`open-questions`、`artifact-diff` 与 `artifact-` 版本记录联动。
- 收敛外部材料反哺路径：同事邮件/会议纪要等作为 `source-ref` 先入账，非授权不入长期记忆，避免污染当前上下文。
- 进一步强化 Runtime / SQLite：每次主控回合固定写入 Stage、SOP、Skill、Context Packet、Invocation、Review Item、Artifact Version、Gate 决策与事件指标。
- 强化分批评审与角色优先级：评审角色按 `required / triggered / optional` 分批召唤，不允许容量上限导致关键角色丢失。
- 回归测试扩展到“评审可见性 / 分批/复评逻辑 / 非产品任务退出流程”的覆盖。
