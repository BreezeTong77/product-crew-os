# Deep Artifact Pack

## 1. 使用场景

当最小 artifact 不足以支持评审、对齐、立项、交付或上线时，主控教练应触发 Deep Artifact Pack。

## 2. 触发条件

- 阶段门被阻塞。
- 多个 stakeholder 对同一产物有分歧。
- 用户需要拿给老板、客户、研发、设计、数据或测试正式对齐。
- 当前产物会进入排期、上线、采购、合同或外部承诺。

## 3. Pack 类型

| 阶段 | Deep Artifact Pack |
| --- | --- |
| 商业论证 | business-case-pack |
| 用户研究 | research-pack |
| 问题定义 | problem-framing-pack |
| 方案设计 | solution-design-pack |
| PRD | prd-pack |
| 技术评审 | technical-pack |
| 验收测试 | qa-pack |
| 上线 | launch-pack |
| 复盘 | postmortem-pack |

## 4. 最小组成

- 背景与目标
- 当前阶段
- 关键结论
- 证据与来源
- 决策记录
- 争议点
- 风险与阻塞
- 子 Agent 评审摘要
- 采纳 / 拒绝 / 待确认清单
- 下一步行动

## 5. 输出要求

- 中文可阅览。
- Markdown/YAML/JSON 作为源文件。
- Word/PDF 只能作为导出物。
- 所有真实项目材料必须留在 Project Workspace，不得进入公共产品规则。
