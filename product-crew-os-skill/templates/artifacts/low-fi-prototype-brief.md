## 5. 需要验证的问题

- 用户是否知道从哪里开始。
- 用户是否理解下一步。
- 用户是否能完成核心任务。
- 用户是否知道失败后怎么办。

## 6. 推荐生成路径

默认按“先看见、再能点、最后进入用户常用设计工具”的顺序推进：

1. 生成 image 概念图：用于快速确认页面气质、布局方向和关键屏幕，不作为最终需求结论。
2. 生成 HTML Demo：用于验证点击路径、状态反馈、文案和核心交互。
3. 对接 Pencil / Figma：在用户明确授权后，通过 MCP / 插件 / 导入文件把原型转成可编辑设计稿。

如果用户只需要低保真验证，可以停在页面状态表或 HTML Demo。
如果用户希望更高保真，优先走 `image -> HTML Demo -> Pencil/Figma`。

## 7. 应召唤角色

- 产品设计：检查路径、信息层级和状态反馈。
- 技术负责人：仅在组件、交互或数据状态影响实现时进入。
- 客户成功 / CS：检查上线后用户是否真的会用。
- 客户（老板）：仅在需要客户验收、客户演示或采购决策时进入。
