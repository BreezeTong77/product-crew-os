# Workflow Map

Use this map to decide what 主控产品教练 should do next.

For detailed stakeholder boundaries, use `stage-boundary-matrix.md`. This file is the simple navigation map; the boundary matrix is the authority map.

| Stage | User Signal | Required Artifact | Stage Gate | Common Skill |
| --- | --- | --- | --- | --- |
| Opportunity discovery | “我发现一个方向” | opportunity note | problem has a target user | opportunity mapping |
| User research | “我访谈完了” | interview summary | evidence has user quotes | interview summary |
| Problem framing | “我不知道客户到底要什么” | problem statement | problem separated from solution | problem framing, JTBD |
| Requirement analysis | “这个需求怎么拆” | scope and assumptions | MVP boundary is clear | user stories, acceptance criteria |
| Solution design | “方案怎么做” | flow or solution brief | core path is testable | compare solutions |
| PRD drafting | “帮我写 PRD” | PRD v0 | reviewer questions are visible | generate PRD |
| Cross-functional review | “帮我评审” | review notes | blockers are named | review PRD |
| Delivery planning | “准备开发” | task plan | dependencies are explicit | task breakdown |
| Launch readiness | “准备上线” | release checklist | risk owner assigned | release checklist |
| Post-launch review | “上线后怎么样” | postmortem | learning captured | postmortem |

If the current stage is uncertain, 主控产品教练 should state the likely stage and propose the next smallest useful action.
