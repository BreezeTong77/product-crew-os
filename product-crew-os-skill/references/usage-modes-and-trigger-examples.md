# 使用模式与触发语

本文件用于新用户 onboarding、README 展示和主控教练回答“我该怎么用”。

## 1. 三种使用模式

### A. 单点能力调用

适合用户只想完成一个具体任务。

示例：

```text
帮我判断这个需求是真需求还是伪需求。
帮我把这段会议纪要提炼成决策和 action item。
帮我给这个 PRD 做一次内审。
帮我生成低保真原型 brief。
```

主控教练应：

- 识别最小任务。
- 选择一个主要 skill 或模板。
- 输出可编辑 artifact。
- 提醒是否需要进入完整流程。

### B. 完整工作流推进

适合用户从 0 到 1 做一个产品、需求或版本。

示例：

```text
帮我从一个想法走到 PRD。
帮我从商业论证开始，完整跑一遍产品流程。
帮我做一个新功能，从需求验证到任务拆解。
```

主控教练应：

- 建立 Project Workspace。
- 生成 project card。
- 判断当前阶段。
- 逐阶段推进：问题 -> 证据 -> 方案 -> 原型 -> PRD -> 评审 -> 任务 -> 验收 -> 上线。
- 每个阶段结束时输出状态栏和下一步。

### C. 中途插入

适合用户项目已经进行中，只是在某个点卡住。

示例：

```text
我 PRD 写一半了，帮我看现在缺什么。
客户临时提了个需求，帮我判断要不要插进本期。
研发说这个做不了，帮我组织一次技术预审。
这个原型给客户看之前，帮我拉文设计和黑老板看一眼。
```

主控教练应：

- 先诊断当前阶段。
- 盘点已有 artifact。
- 插入最小必要 skill 或 stakeholder review。
- 把结果写回当前项目，而不是重开一套流程。

## 2. 推荐触发语

| 用户意图 | 推荐触发语 | 可能阶段 |
| --- | --- | --- |
| 新想法 | 帮我判断这个想法值不值得做 | project_intake, value_sizing |
| 商业判断 | 从商业论证开始跑一遍 | business_context |
| 真伪需求 | 判断这是不是真需求 | problem_definition, evidence_inventory |
| 用户研究 | 帮我设计访谈问题 | research_plan, interview_guide |
| 方案设计 | 给我 2-3 个可行方案 | solution_exploration |
| 原型 | 帮我做低保真原型 | low_fi_prototype |
| PRD | 帮我写 PRD v0 | prd_v0_draft |
| 内审 | 帮我做产品自审 | pm_self_review |
| 评审 | 拉业务和技术评审一下 | internal_product_review |
| 任务拆解 | 拆成 Epic / Story / Task | task_breakdown |
| 验收 | 生成验收标准和测试场景 | acceptance_criteria |
| 上线 | 生成上线检查清单 | launch_readiness |
| 复盘 | 帮我做项目复盘 | post_launch_review |

## 3. 主控教练话术

当用户不知道怎么开始：

```text
你可以直接说目标，不用选 skill。
比如：“帮我从一个想法走到 PRD”，我会先判断阶段，再决定该做商业判断、需求验证、原型、PRD 还是评审。
```

当用户想快速完成一个任务：

```text
可以，我们按单点能力跑。我会先生成一个可编辑 artifact，再告诉你要不要接入完整产品流程。
```

当用户中途插入：

```text
我先不重开流程。先判断你现在卡在哪个阶段，再把需要的评审或模板插进去，处理完再回到主流程。
```
